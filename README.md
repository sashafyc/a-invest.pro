## Воркфрейм: написание новой статьи

### Шаг 1 — выбрать статью
`_materials/content-plan.md` → ⬜ статья с 🔴 приоритетом.

### Шаг 2 — подготовить материал
- Прочитать TG-посты по теме (`_materials/tg-posts.txt`)
- Выписать цифры, кейсы, формулировки Александра

### Шаг 3 — написать по шаблону
Скопировать `_template.html` → `blog/SLUG.html`. Следовать инструкции (ШАГ 1–5).

### Шаг 4 — чеклист публикации
- [ ] Карточка добавлена в `blog.html`
- [ ] URL добавлен в `sitemap.xml`
- [ ] Перелинковка в 1–2 существующих статьях
- [ ] Список статей обновлён в `_template.html`
- [ ] Статус в `_materials/content-plan.md` → ✅
- [ ] README.md обновлён (счётчик статей + список)
- [ ] `_materials/brief.md` обновлён (актуальные задачи)
- [ ] Коммит и пуш

### Шаг 5 — предложить дистрибуцию
Если статья есть в `distribution-tracker.md` (не `—` во всех колонках) — сразу после публикации написать:
> «Статья опубликована. Эта тема подходит для дистрибуции — сделать версии для [площадки]?»

После «да» — создать файлы в `_distributions/SLUG/`, обновить трекер (`🗂`), обновить счётчики.

## Воркфрейм: дистрибуция

**Цикл:** каждые ~5 новых статей → делаем дистрибуции по накопившимся.

**Форматы:** подробно в `_materials/distribution-guide.md`
- `vc.html` — VC.ru, 50–60%, профессиональный тон
- `dzen.html` — Яндекс Дзен, 60–70%, личный голос/storytelling
- `tenchat.txt` — TenChat, 400–600 символов, тезис + буллеты + ссылка

**Как публиковать:** открыть `.html` в браузере → Cmd+A → Cmd+C → вставить в редактор площадки.

**Статусы трекера:**
- `⬜` — нужна, файл не готов
- `🗂` — готово, не выложено (**напоминать каждую сессию**)
- `✅` — выложено

Когда пользователь говорит «выложил [площадку/статью]» — обновить трекер `🗂 → ✅`.
