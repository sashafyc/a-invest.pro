# Гайд по дистрибуции контента

## Концепция

Каждая новая статья сразу идёт в дистрибуцию на **6 площадок**.
Цель: внешний охват и трафик обратно на сайт. НЕ публикуем полные копии — только адаптированные версии со ссылкой на оригинал.

Папка: `_distributions/` (в .gitignore — не деплоится на сайт)

## Форматы по платформам

### VC.ru (`vc.html`)
- **Объём:** 50–60% от оригинала
- **Угол:** профессиональный, аналитический. Факты, цифры, практика.
- **Как публиковать:** открыть `.html` в браузере → Cmd+A → Cmd+C → вставить в редактор VC.ru
- **Финал:** ссылка «Читать полностью» + ссылка на express-ocenka.html + подпись Александра
- **Тема:** указывать обязательно — влияет на охваты. Тестируем разные (Личный опыт, Деньги, Инвестиции)
- **Лимит:** 1 статья в месяц на бесплатном тарифе
- **SEO:** на бесплатном тарифе не индексируется поисковиками — ценность в охвате, не в ссылках

### Дзен (`dzen.html`)
- **Объём:** 60–70% от оригинала
- **Угол:** личный, storytelling. «Я видел», «У меня был клиент», «Расскажу честно».
- **Заголовок:** другой, не как у статьи на сайте (указать в комментарии файла)
- **Как публиковать:** открыть `.html` в браузере → Cmd+A → Cmd+C → вставить в редактор Дзен
- **Финал:** ссылка на полную статью + ссылка на express-ocenka.html + подпись
- **Важно:** Дзен любит личные истории, первое лицо, конкретные случаи из практики

### TenChat (`tenchat.txt`)
- **Объём:** 400–600 символов
- **Формат:** plain text, без HTML-тегов
- **Структура:** 1 тезис → буллеты через тире (без нумерации!) → ссылка → хэштеги
- **Хэштеги:** #бизнес #продажабизнеса #предпринимательство #брокер #инвестиции
- **Как публиковать:** вставить в TenChat editor → Escape после каждого хэштега (автодополнение!)
- **Важно:** не нумеровать пункты — TenChat добавит свою нумерацию, будет задвоение

### Spark.ru (`spark.html`)
- **Объём:** 50–60% от оригинала
- **Угол:** инвестиционный / стартап-бизнес. Аудитория — инвесторы, предприниматели, стартаперы.
- **Чередование:** статья 1 — от лица компании (ООО А-Инвест), статья 2 — личная (Александр Неделюк), и так далее
- **Финал:** ссылка на полную статью + подпись
- **Как публиковать через Chrome-автоматизацию:**
  1. Пользователь СНАЧАЛА логинится вручную (auth → chrome-extension, Claude не может)
  2. Пользователь открывает `spark.ru/write/1/COMPANY_ID` или `/write/2/PERSONAL_ID`
  3. Claude заполняет ВСЁ через JavaScript за один проход:
     - `input[name="title"]` — заголовок + `.dispatchEvent(new Event('input', {bubbles:true}))`
     - `textarea[name="subtitle"]` — вступительный абзац (320 знаков)
     - Clipboard API для тела: `navigator.clipboard.write([new ClipboardItem({'text/html': blob})])` → клик в body → Cmd+V
     - `input[name="tags"]` — теги через запятую
  4. Публикация: `document.querySelector('.post-form__submit').click()`
  5. **КРИТИЧНО:** поле title должно быть заполнено, иначе кнопка публикации молча не работает (класс `missed`)
- **Проблемы:**
  - Сессия Spark протухает ОЧЕНЬ быстро (минуты). Публиковать СРАЗУ после логина
  - Авторизация через chrome-extension → Claude не может логиниться сам
  - Не кликать в body для ввода тегов — попадёт в текст статьи. Заполнять теги ТОЛЬКО через JS

### Habr (`habr.html`)
- **Статус:** пока публикуем в **песочницу** (sandbox), ждём инвайта модератора
- **Объём:** 70–80%, развёрнуто и технично
- **Угол:** аналитический, экспертный. Термины, методология, цифры. Хабр не любит воду и рекламу.
- **Подходящие темы:** due diligence, оценка бизнеса, финансовая отчётность, юридика сделок
- **Как публиковать:** вставить в редактор Habr вручную (нет copy-paste из HTML)
- **Финал:** ссылка на статью как «источник» или «подробнее» — ненавязчиво
- **Задача:** получить инвайт — писать качественные статьи в песочницу + активность в комментариях

### Telegraph (`telegraph.html`)
- **Объём:** полная копия статьи или 80–90%
- **Угол:** тот же что на сайте — это зеркало для SEO
- **Как публиковать:** telegra.ph → новая страница → вставить контент → опубликовать
- **Финал:** ссылка «Оригинал: a-invest.pro/blog/SLUG.html»
- **SEO:** страницы Telegraph индексируются Яндексом и Google — даёт дополнительный URL под те же ключи

## Уроки Chrome-автоматизации (обновлять по мере опыта)

### Общие правила
- **Clipboard API** работает: `navigator.clipboard.write([new ClipboardItem({'text/html': blob})])` → клик в editor → Cmd+V
- **"Document is not focused"**: перед clipboard.write() нужно сначала кликнуть в редактор
- **Не вводить текст кликом + type в поля рядом с editor body** — текст может попасть в body. Использовать JS `.value` для input/textarea
- **После вставки контента — сразу публиковать**, не переключаться между полями мышкой

### По площадкам: что работает через Chrome
| Площадка | Логин | Заполнение | Публикация | Примечания |
|----------|-------|------------|------------|------------|
| Дзен | ✅ уже залогинен | Clipboard HTML paste | ✅ через UI | Работает стабильно |
| TenChat | ✅ уже залогинен | Type text | ✅ через UI | Не нумеровать (задвоение) |
| Spark | ❌ chrome-extension auth | JS fill all fields | JS click `.post-form__submit` | Сессия протухает быстро! |
| Telegraph | ✅ без авторизации | Clipboard HTML paste | ✅ через UI | URL нельзя поменять после публикации |
| Habr | ? не проверено | ? | ? | Песочница |
| VC.ru | ✅ уже залогинен | Clipboard HTML paste | ✅ через UI | 1/мес лимит |

### Порядок публикации (от простого к сложному)
1. **Telegraph** — без авторизации, самый простой
2. **TenChat** — просто вставить текст
3. **Дзен** — clipboard paste, стабильно
4. **VC.ru** — clipboard paste (если месяц не исчерпан)
5. **Habr** — ручная вставка в песочницу
6. **Spark** — ПОСЛЕДНИМ, т.к. сессия протухает. Пользователь логинится → Claude сразу заполняет и публикует

## Воркфрейм дистрибуции (на каждую статью)

1. Создать папку `_distributions/SLUG/`
2. Написать все 6 файлов
3. Обновить трекер → `🗂` по всем площадкам
4. Опубликовать через Chrome: Дзен → TenChat → Spark → Telegraph → VC (1/мес) → Habr (песочница)
5. Обновить трекер → `✅` по каждой выложенной
6. **Удалить папку** `_distributions/SLUG/` — файлы отработали, статусы хранятся в трекере
